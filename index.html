Git Start!

常用命令

Git

Git 基本操作

  1. git init 选择一个空目录初始化
  2. 创建指定目录learngit并加入readme
  1. git status 查看添加到工作区与暂存区的内容是否一致
  2. 当在本地发生了修改可以通过git diff查看修改的变化
    $ git diff readme.txt
  3. 提交所有暂存区的内容到仓库+提交说明（方便管理）
    $ git commit -m "wrote a readme file"
  4. 可以使用git log查看提交的日志，通过commit后的版本号回退版本
    commit 461a4c83a94bc47621c873d30b9165db9e48c78d
    • 查看日志图表
      $ git log --graph
    • 查看简洁版日志图表
      $ git log --graph --pretty=oneline --abbrev-commit
  5. 回退
    • 回退暂存区的特定文件（暂存区->工作区）
      $ git reset -- filename
    • 回退到前一个版本
      $ git reset --hard HEAD^
    • 回退到特定版本（使用版本号不发生重复的前n位>3）
      $ git reset --hard 461a
    • 查询恢复版本的记录
      $ git reflog
    • 撤销工作区的修改（暂存区->工作区）
      $ git checkout -- readme.txt
    • 撤销暂存区的修改（仓库->暂存区）
      $ git reset HEAD readme.txt
  6. 删除文件使用git rm将修改保存到暂存区
    $ git rm readme.txt
  7. 连接本地仓库到GitHub
    • 创建SSH Key
      $ ssh-keygen -t rsa -C "youremail@example.com"
    • 将id_rsa.pub 添加到GitHub

      登陆GitHub->“Account settings”->“SSH Keys”添加“SSH Key”，任意Title，在Key文本框里粘贴id_rsa.pub文件的内容

    • GitHub上建立新仓库learngit
    • 本地与在线仓库关联
      $ git remote add origin git@github.com:arnoldteams/learngit.git

      远程库的名字是origin
      关联仓库为arnoldteams/learngit

    • 将本地推送到GitHub仓库
      $ git push -u origin master
      推送失败可以先进行合并
      $ git pull --rebase origin master
  8. 连接完成后若本地仓库发生修改进行推送
    $ git push origin master
  9. GitHub 仓库克隆到本地
    $ git clone git@github.com:arnoldteams/learngit.git
    或者
    $ git clone https://github.com/Arnoldteams/learngit

分支

  1. 初始只有master分支

  2. 添加dev分支

    • 创建及切换分支

    • 查看所有分支git branch

  1. 向dev提交
  2. 合并分支
    • 合并dev到当前分支
      $ git merge dev
    • 保留新分支进行合并
      $ git merge --no-ff -m "merge with no-ff" dev

  1. 删除分支
    $ git branch -d dev
